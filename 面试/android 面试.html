  • BroadcastReceiver
    • 启动方式
      • 隐式启动
        • 在 Mainfest.xml 中注册启动
      • 显式启动
        • 在 java 代码中注册启动
        • 记得要 unregisterReceiver
    • 分类
      • 全局广播
      • 局部广播
        • 使用的类
          • LocalBroadcastManager
            • LocalBroadcastManager.getInstance(context);
            • LocalBroadcastManager.setBroadcast(intent)
            • LocalBroadcastManager.registerReceiver(LocalReceiver, intentFilter);
            • LocalBroadcastManager.unregisterReceiver(LocalReceiver)
          • LocalReceiver
        • 本地广播不能通过静态注册的方式来接收
    • 优先级分
      • 普通广播
        • sendBroadcast()
      • 有序广播
        • sendOrderedBroadcast()
        • 在 Manifest.xml 文件中设置 android:priority= 优先级，数值越大，优先级越高，首先接收到广播
        • 如果在 onReceive() 方法内调用了 abortBroadcast() 拦截了广播，则后续的接收者，不会受到广播
    • 不能在 onReceive() 方法里面执行耗时动作， 参考 10s 以内
      • 不然会导致 ANR
      • 如果一个进程里面只存在一个 BroadcastReceiver组件, 并且在其中开启子线程执行耗时任务, 系统会认为该进程是优先级最低的空进程, 很容易将其杀死
      • 如果要执行耗时任务，则应该启动一个 Service
    • 工作过程 （见博客 《四大组件的工作过程》https://blog.csdn.net/yxhuang2008/article/details/51439344）
    • 注意点
      • Android 5.0 中，默认情况下，广播不会发送给已经停止的应用
      • 从 Android 3.1 开始，处于停止状态的应用无法接收到开机广播，而 3.1 之前，处于停止状态的应用是可以收到开机广播的
      • 如果想要已经停止的应用接收广播，需添加 FLAG_INCLUDE_STOPPED_PACKAGES

    • 生命周期
      • onAttch -> onCreate -> onCreateView -> onActivityCreated -> onStart -> onResume -> onPause -> onStop -> onDestroyView -> onDestory -> onDetach
    • 通信问题
      • Fragment 与 Activity 之间的通信
        • setArg
      • Fragment 与 Fragment 之间的通信
        • 回调
    • hide 与 replace 方法
      • hide 只是把 fragment 隐藏
      • hide 和 show 一起调用， 会调用 setVisibility 方法，而不会调用生命周期函数
      • replace 则是替换掉
      • add 和 replace 一起调动，会调用 fragment 的 onDestoryView、onCreateView等一系列生命周期
    • onHiddenChanged的回调时机
      • 当使用add()+show()，hide()跳转新的Fragment时，旧的Fragment回调onHiddenChanged()，不会回调onStop()等生命周期方法，而新的Fragment在创建时是不会回调onHiddenChanged()

    • AsyncTask
      • 封装了 线程池和 Handler, 不适合进行特别耗时的后台任务
      • 四个核心方法
        • onPreExecute() 主线程中执行
        • doInBackground(Params...params) 线程池中执行
        • onProgressUpdate(Progress...values) 在主线程中执行
        • onPostExecute(Result result) 主线程中执行
      • 注意点：
        • AsyncTask 的类必须在主线程中执行，因为内的静态对象 sHandler 是用来切换到主线程的
        • AsyncTask 的对象必须在主线程中创建
        • execute 方法必须在 UI 线程调用
        • 在 Android 1.6 之前， AsyncTask 是串行执行的，1.7 - 2.9 版本是并行执行，3.0 之后是串行执行，可以调用 executeOnExecutor 并行执行任务
